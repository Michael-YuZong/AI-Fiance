# 2026-03 Change Log

这份文件是按需阅读的历史归档，不是默认入口。

只有在需要追某个判断为什么变、某个合同是什么时候收口、或者某个 backlog 为什么迁移时再读。

## 2026-03-21

### `strategy` benchmark fixture v1

- `predict / replay` 现在会显式写出 `benchmark_fixture`
- fixture 会披露资产窗口、基准窗口、overlap 行数、`as_of` 是否对齐、未来 `20d` 验证窗是否 ready
- `validate / experiment` 摘要也会汇总这层 fixture
- 如果 benchmark overlap 不足或 `as_of` 未对齐，主预测会退回 `no_prediction`

### proxy signals 进入 `briefing / research / retrospect`

- `briefing` 会在正文、payload 和 final manifest 里正式带出 `proxy_contract`
- `research` 新增统一的 `代理信号与限制` 段落
- `portfolio review / retrospect` 会把交易当时保存下来的 `proxy_contract` 写进 `decision_snapshot`
- 旧交易如果没有历史代理快照，不会伪造 point-in-time 代理判断

### proxy signals 进入 pick 正式合同

- `stock_pick / etf_pick / fund_pick` 的客户稿现在显式写出 `市场风格代理 / 情绪代理`
- pick final 的 manifest 已补进 `proxy_contract`
- `review_audit` 开始审这层合同

## 2026-03-20

### 客户稿先给结论，再给解释

- `scan / stock_analysis / stock_pick / etf_pick / fund_pick` 的 major section 会统一补一行 `先看结论`
- `观察 / 回避` 口径动作区新增 `触发买点条件`
- 观察稿还会补 `关键盯盘价位`

### `stock_pick` 客户详细稿分层

- `正式推荐` 继续保留完整八维、催化、硬检查和风险拆解
- `看好但暂不推荐 / 观察为主` 压成短版卡片
- 观察名单同时保留 `1` 只代表样本详细拆解，满足 final 门禁

- `基本面/催化主导` 路径把技术门槛从 `40` 放宽到 `35`
- `趋势延续` 路径把门槛从 `tech>=60 / relative>=70 / risk>=55 / catalyst>=35` 放宽到 `tech>=55 / relative>=65 / risk>=45 / catalyst>=25`

### 推荐口径去“只会观察”

- `stock_pick / etf_pick` 默认给出 `短线优先 + 中线优先`
- A 股个股推荐拆成 `核心主线` 和 `低门槛可执行 + 关联ETF平替`
- `etf_pick` payload 会拆 `short_term / medium_term`

### ETF / 基金推荐合同补强

- 低覆盖率稿件会真正退成 `摘要观察稿`
- `回避 / 观察` 结论下会裁掉完整交易动作表
- ETF / 基金维度表的 `基本面` 改按 `产品质量/基本面代理` 披露
- 正式报告外审合同补成双层：`rich prompt 结构化审稿 + 零提示发散审`
- `etf_pick / fund_pick` 的客户稿区分 `配置视角` 和 `交易视角`
- ETF/基金催化面新增 `产品/跟踪方向催化`
- 动作区新增 `建议买入区间` 和 `建议减仓区间`

### 个股推荐校准

- `macro_reverse` 不再机械把所有原本能到 `⭐⭐⭐` 的个股统一压回 `⭐⭐`
