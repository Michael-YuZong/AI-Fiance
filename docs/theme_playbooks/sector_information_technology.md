# 信息技术行业认知卡

## 市场通常在交易什么

- 市场在交易的是产业资本开支、产品周期、国产替代进度和风险偏好，而不是一个笼统的“科技成长”口号。
- 软件、硬件、半导体、互联网和通信设备都属于信息技术，但驱动节奏、估值容忍度和兑现路径并不一样。

## 常见正向驱动

- 新一轮资本开支、产品创新、AI 应用扩散、国产替代推进和企业 IT 投入恢复。
- 龙头订单、出货、渗透率和盈利弹性被同步验证。

## 常见反向风险

- 叙事很强，但订单、商业化和利润兑现跟不上。
- 估值已经提前打满，边际利好不足以支撑继续扩张。

## 首页应优先联想到的宏观 / 产业变量

- 风险偏好、美元利率、资本开支、企业 IT 预算、产品周期、渗透率、出货和库存水平。

## 典型传导链

- 流动性与风险偏好改善 / 产业资本开支启动 -> 订单和出货改善 -> 龙头盈利与估值抬升 -> 细分方向轮动扩散。

## 常见所处阶段

- 更常见的是“叙事预热 -> 订单验证 -> 业绩兑现 -> 细分赛道分化”，而不是所有科技方向一起上涨。

## 轮动与拥挤度

- 信息技术内部切换很快，常在半导体、算力、软件应用和终端链之间轮动；不要把科技都写成同一条主线。

## 证伪信号

- 如果资本开支、订单、出货和利润率没有形成接力，或者只剩高估值叙事没有兑现，这条行业逻辑就不能继续按顺风写。

## 不能误写成直接催化的东西

- “科技成长长期空间很大”不能直接写成当天催化。
- 产业政策支持和概念热度不能替代真实订单、产品发布和商业化验证。

## 写首页时应避免的模板化表达

- 不要只写“科技板块弹性大”。
- 要说明今天交易的是资本开支、产品周期、国产替代，还是风险偏好修复。
