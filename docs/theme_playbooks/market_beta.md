# 宽基 / 市场Beta

## market_logic
- 这类标的更像在交易市场整体风险偏好、风格切换和宽基轮动，而不是单一产业链事件。
- 同样是宽基，A500、沪深300、中证500、中证1000、创业板和科创50 对应的其实是不同风格 beta，不该混成“无主题”。

## bullish_drivers
- 流动性改善、风险偏好回升、宽基扩散、权重股盈利预期上修，都可能带来顺风。
- 高低切、风格切换和增量资金回流，是宽基 beta 最常见的上行动力。

## risks
- 最常见的问题不是“主题失效”，而是风格错配：你买的是宽基，但市场在交易另一种风格。
- 如果市场只是少数龙头抱团、宽度不足，宽基很容易看起来稳，但实际缺乏弹性。

## variables
- 优先联想到市场宽度、量能、风格偏好、权重股盈利、风险偏好和增量资金来源。
- 对创业板 / 科创50 这类宽基，要额外想到成长风格、估值水位和波动承受能力。

## transmission_path
- 宏观和流动性先改写风险偏好，再通过风格轮动和市场宽度传导到宽基表现，最后才体现在指数级别涨跌。

## stage_pattern
- 更常见的是“风险偏好修复 -> 风格扩散 -> 宽基跟涨”或“权重修复 -> 宽基企稳 -> 题材扩散”，而不是单日消息直接决定长期方向。

## rotation_and_crowding
- 观察宽基时重点看：现在是大票占优、小票补涨、成长修复，还是防御抱团；不同宽基的顺风条件完全不同。

## falsifiers
- 如果量能收缩、宽度走弱、风格切回对立方向，宽基 beta 逻辑就不能继续按顺风写。

## guardrails
- 不要把宽基写成“没有主题”；宽基本身就是风格和市场结构的主题表达。
- 不要把单一成分股事件直接放大成整条宽基逻辑，除非它确实代表权重风格切换。

## style_notes
- 首页应先讲市场环境、板块轮动和风格 beta，再讲这只宽基为什么在当前轮动里顺风或逆风。
