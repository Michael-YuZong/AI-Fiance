# Theme Playbooks

这份文档只做一件事：告诉 `editor subagent` 为什么需要一层“主题认知”，以及这层认知应该怎么用。

## 作用

- 让首页先像研究员一样回答“这个板块今天到底在交易什么”
- 给宏观面、板块轮动、情绪热度和微观判断提供固定认知底座
- 避免把板块常识、产业链逻辑和当天可验证事实混写成一团

## 当前状态

- 当前是 `v1` 可用状态，不是独立收口里程碑
- 已接进 `scan / stock_analysis / stock_pick / etf_pick / fund_pick / briefing` 的首页判断层
- 现在更像 repo 级共享能力，而不是独立的 `theme_analysis` 命令
- 现在除细主题卡外，也支持一层 `sector fallback`，主行业层 broad sector 已基本补齐，避免宽行业标的只有硬标签没有行业叙事
- `sector fallback` 现在还会显式带出 repo 内已定义的 `sector -> subtheme bridge`，并按 `day_theme / notes / headline` 的当前线索动态排序，提醒首页先按行业层判断，再决定是否下钻到细主题
- `bridge` 现在还带 `confidence / reason` 合同：弱信号只允许写成观察方向，强信号最多写成“更偏向某条细分线”，不能把行业层稿件直接偷改成已确认细主题
- 主题匹配现在还带第一版 `conflict guard`：像 `半导体 / 先进封装 / AI算力`、`创新药 / BD出海 / 临床催化 / CXO`、`华为链 / 苹果链 / 英伟达链 / 事件产业链`、`消费复苏 / 白酒 / 家电 / 免税文旅 / 大众消费`、`政策驱动 / 数据要素 / 中特估` 这类易混组如果当前线索还没拉开，会先退回行业层，不把正文硬写成单一细主题
- 这层冲突诊断现在不只出现在 `theme_lines`，`scan / stock_pick / etf_pick / fund_pick / briefing` 的 `micro_lines` 也会显式写“哪几条线还在打架”，避免 editor 只看候选解释层时漏掉这条边界
- 客户稿正文里的 `执行摘要 / 今日动作摘要` 现在也会带 `主题边界 / 细分观察` 行：即使读者已经翻过首页，正文表格里也还能看到“当前先按行业层理解”还是“可以优先留意哪条细分线”
- 客户稿正文里的 `为什么这么判断 / 为什么先看它 / 重点观察` 这类解释段现在也会复用同一条边界提示，不再出现“首页和摘要表都很谨慎，正文解释段却像已确认单一主题”的错位
- `client-final` 的 `release_manifest` 和 `review_audit` 现在也开始带 `theme_playbook_contract`：后续外审可以直接追踪这份稿到底是命中了细主题、退回行业层，还是处在 `ambiguous_conflict / sector bridge` 边界，不必只靠翻正文找线索
- `release_check` 现在也会先看首页判断和正文有没有把同一条边界写完整：如果首页已经写了“行业层冲突”或“优先留意某条细分线”，但正文没有落出 `主题边界 / 细分观察`，会先作为 pre-release finding 暴露出来，不等到 `report_guard` 才拦
- 这层 `release_check` 现在还会直接对照 `editor_payload.theme_playbook` 和 `editor_prompt.md`：如果 sidecar 已明确命中某个主题、或已给出 `ambiguous_conflict / sector bridge` 的具体候选和下钻主线，但客户稿正文没有把这些具体标签带出来，也会先报 pre-release finding，避免 sidecar 和成稿两套口径
- `report_guard` 现在也会对这份合同做第一版正文一致性校验：如果 manifest 已说明“当前还是行业层冲突稿”或“已经给出 sector bridge 的中高置信度下钻方向”，但正文完全没写出 `主题边界 / 细分观察`，正式交付会直接被拦下

## 使用方式

生成首页时，优先把 `主题认知卡` 和 `当天市场上下文` 一起喂给 editor。

推荐顺序：

1. 先识别标的属于哪个主题
2. 读取对应 playbook
3. 再结合当天行情、轮动、情绪、催化和标的底稿写首页

## 固定结构

每张主题认知卡建议包含下面 10 部分：

1. 市场通常在交易什么
2. 常见正向驱动
3. 常见反向风险
4. 首页应优先联想到的宏观 / 产业变量
5. 典型传导链
6. 常见所处阶段
7. 轮动与拥挤度
8. 证伪信号
9. 不能误写成直接催化的东西
10. 写首页时应避免的模板化表达

## 写作原则

- 主题认知不是当天新闻摘要
- 主题认知不是历史科普
- 主题认知不是打分器
- 主题认知只负责回答“这一类标的通常该先想到什么”
- 首页写作时，不能把 playbook 里的共识直接写成当天已验证事实
- 如果未来要落成具体卡片，可以按 `半导体 / 先进封装 / AI算力 / 机器人 / 自动驾驶 / 能源资源 / 恒生科技 / 创新药 / 政策驱动 / 事件产业链` 继续补充，但这份 README 本身只定义用法，不绑定具体实现文件

## 分类原则

首页写作默认先分两层：

1. 标准行业分类（硬分类，尽量互斥）
2. 主题 / 概念分类（软分类，可以重叠）

行业分类用于回答“它大体属于哪一类资产暴露”，主题分类用于回答“市场这几天真正围绕什么在交易”。

推荐先按下面的层级理解：

- 行业层：信息技术、医疗保健、金融、可选消费、必选消费、工业、材料、能源、公用事业、房地产、通信服务
- 主题层：
  - 技术路线：AI算力、机器人、自动驾驶、低空经济、卫星互联网、量子计算、固态电池、钙钛矿、先进封装、HBM、Chiplet
  - 政策驱动：国产替代 / 信创、央企市值管理、新质生产力、数据要素、碳中和 / 绿电、一带一路、专精特新
  - 周期 / 宏观：高股息 / 红利、中特估、黄金 / 有色资源、煤炭 / 能源安全、地产链复苏、消费复苏
  - 事件产业链：华为链、苹果链、英伟达链、特斯拉链
