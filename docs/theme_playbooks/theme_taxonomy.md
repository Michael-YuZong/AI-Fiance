# Theme Taxonomy

这份文档不直接写首页，而是给 `theme playbook / editor payload` 定义一套更稳定的“行业 + 主题”坐标系。

## 1. 标准行业分类（硬分类，尽量互斥）

默认优先靠近下面这套大类：

- 信息技术
- 医疗保健
- 金融
- 可选消费
- 必选消费
- 工业
- 材料
- 能源
- 公用事业
- 房地产
- 通信服务

在 A 股语境里，可以近似映射到申万一级行业；在跨市场或 ETF 语境里，优先贴近 GICS 大类。

行业分类回答的是：

- 这只标的的大类暴露是什么
- 它更像成长、周期、防守还是金融
- 它应该先和谁比较，而不是和所有主题混在一起

## 2. 主题 / 概念分类（软分类，可以重叠）

主题分类回答的是：

- 市场这几天到底在交易什么
- 这只标的为什么会受这条叙事影响
- 这条叙事是长期主线、短线事件，还是阶段性脉冲

### 技术路线类

- AI算力：光模块、服务器、交换机、算力租赁、液冷
- 半导体：晶圆、设备、材料、存储、先进封装、HBM、Chiplet
- 机器人：人形机器人、伺服、减速器、运动控制
- 自动驾驶：智驾、车路云、激光雷达、线控底盘
- 低空经济：eVTOL、通航、飞控、空域管理
- 卫星互联网：卫星制造、地面站、卫星通信
- 量子计算
- 固态电池
- 钙钛矿

### 政策驱动类

- 国产替代 / 信创
- 央企市值管理
- 新质生产力
- 数据要素
- 碳中和 / 绿电
- 一带一路
- 专精特新

### 周期 / 宏观类

- 高股息 / 红利
- 中特估
- 黄金 / 有色资源
- 煤炭 / 能源安全
- 地产链复苏
- 消费复苏

### 事件产业链类

- 华为链
- 苹果链
- 英伟达链
- 特斯拉链

## 3. 写首页时怎么用

建议先回答这 3 个问题：

1. 它属于哪个行业大类
2. 今天市场在交易哪条主题线
3. 行业大类和主题线之间，是共振、错位，还是冲突

不要只写一个“主题名词”，而不说明：

- 这条主题到底靠什么驱动
- 它现在是景气兑现、估值修复，还是事件脉冲
- 这条主题对这只标的是直接影响，还是间接映射
