# Investment Agent Skill

## 概述

个人投资决策辅助系统。输入标的代码，输出多维度结构化打分卡，并逐步扩展到简报、组合、风险和研究场景。

如果用户没有明确输入命令，而是直接用自然语言描述需求，应优先推断到最匹配的命令；也可以直接使用 `assistant <请求>` 自然语言入口。

当本地命令返回空数据、上游源不可用、或明显缺少用户关心的关键信息时，不要停在失败提示上。应按下面顺序继续完成任务：

1. 先尝试本地缓存和备用数据源
2. 仍不足时，允许联网补查
3. 联网时优先官方、一级媒体、交易所、央行、指数公司、基金公司或主流行情源
4. 最终回答里明确区分“程序输出”和“联网补充”

## 晨报正确性校验

生成 `briefing daily` 或任何“今日市场主线”类回答时，不允许直接把 `regime` 当成当天主线。必须区分：

- `背景环境`：例如温和复苏、滞涨、通缩
- `今日主线`：例如能源冲击、利率预期、AI 催化、政策驱动、防守避险

在输出“今日主线”前，至少做下面 4 类交叉校验：

1. `新闻校验`
   - 看新闻分类是否集中在某一条线上，例如 `energy/geopolitics`、`fed`、`earnings`、`ai`、`semiconductor`
2. `价格校验`
   - 看关键资产是否真的共振，例如原油、VIX、DXY、美债收益率、黄金、铜
3. `盘面校验`
   - 看涨停池、强势股池、龙虎榜是否支持该主线，例如电力/电网/油气是否真的在前排
4. `资金校验`
   - 看主力资金、北向板块、热门股是否支持该叙事；若不支持，应降级为“候选主线”

还必须做下面 4 条编辑层校验：

1. `冲突消解`
   - 如果 `背景环境` 和 `今日主线` 打架，默认由 `今日主线` 覆盖日内资产偏好
   - 需要明确写出“背景 regime 暂时降级为中期参考”
2. `异常值防御`
   - 对极端价格变化做 sanity check，例如原油 5 日暴涨、ETF 20 日异常涨跌
   - 命中阈值时必须输出 `⚠️ 数据异常/极端波动，请人工复核`
3. `源覆盖校验`
   - 新闻源少于 2 类，或缺少国内源时，必须显式降级为“主线草稿/覆盖不足”
4. `昨日闭环`
   - 如果存在昨日晨报归档，优先回顾昨日验证点是否兑现
   - 没有归档时也要明确说明“无法自动回顾”，不能跳过不提

晨报成稿还应固定输出这 3 类结构化内容：

1. `驱动与催化`
   - 用事件 -> 传导 -> 交易含义三段式解释今天为什么这样走
2. `今日验证点表`
   - 用表格写清楚“观察点 / 看什么 / 成立意味着 / 失效意味着”
3. `今日日历`
   - 用表格列出当天已知重要时点、事件和潜在影响

如果 `新闻主线` 和 `价格/盘面/资金` 不一致：

- 不要硬写成确定结论
- 应明确写成“候选主线”或“叙事存在冲突”
- 同时说明冲突点在哪里

特别规则：

- 当 `能源/地缘` 新闻集中，且布油单日或 5 日涨幅显著、VIX 上行、A 股强势行业落在电力/电网/油气链时，应优先写成 `能源冲击 + 地缘风险`，而不是让 `regime` 覆盖掉当天事件主线
- 当 `Fed/降息` 新闻存在，但科技代理并未走强、美元和波动率仍偏强时，不应把晨报写成“成长修复日”

## 晨报结构要求

默认结构应贴近“每日晨报模板 v4”，正文按倒金字塔收敛到 5 块，附录折叠：

1. `昨日验证回顾`
   - 优先回答昨天的验证点对了还是错了
   - 没有归档时明确写“无法自动回顾”
2. `主线判断与行动`
   - 先给主线，再给今天怎么做
   - 在同一块里完成 regime 裁决和主线校验
3. `市场全景`
   - 至少包含 `国内市场概览`、`风格与行业`、`宏观资产`、`隔夜外盘`、`Watchlist`
   - Watchlist 正文只保留 `RSI + ADX`
4. `驱动与催化`
   - `核心事件` 用“标题(来源) -> 传导 -> 交易含义”表达
   - `今日日历` 必须拆成 `市场事件` 和 `操作提醒`
   - `盘面与资金` 必须单列，并说明缺失项替代方案
5. `今日验证点`
   - 只保留可证伪的验证点表，必须有量化判断标准
6. `组合与持仓`
   - 无持仓时简明说明即可

附录使用折叠块放：

- 完整技术指标
- 龙虎榜明细
- 资金流代理与情绪
- 股指期货与期权
- 重点公司财报点评（财报季启用）
- 分层配置建议

禁止出现的问题：

- 同一个结论在 3 个以上模块重复出现
- `So What` 被埋到最后
- 只有代码没有上下文，例如 ETF 价格不注明是场内价格/现价/指数点位

## 可用命令

| 命令 | 功能 | 示例 |
| --- | --- | --- |
| `scan <代码>` | 六维度标的扫描 | `scan 561380` |
| `snap <代码>` | 盘中实时快照 | `snap 561380` |
| `compare <代码...>` | 同类标的横向对比 | `compare 518880 518800 159934` |
| `briefing daily` | 每日晨报，可附带新闻源偏好 | `briefing daily --news-source Reuters` |
| `briefing weekly` | 每周周报 | `briefing weekly` |
| `portfolio status` | 查看持仓状态 | `portfolio status` |
| `portfolio thesis` | 查看 / 管理论点 | `portfolio thesis HSTECH` |
| `portfolio log` | 记录操作 | `portfolio log buy QQQM 185.5 10000` |
| `portfolio review` | 月度操作复盘 | `portfolio review 2026-02` |
| `risk report` | 组合风险报告 | `risk report` |
| `risk stress` | 压力测试 | `risk stress "美股崩盘"` |
| `risk correlation` | 相关性矩阵 | `risk correlation` |
| `backtest <规则>` | 规则回测 | `backtest macd_golden_cross HSTECH 3y` |
| `discover` | 主动机会发现 | `discover` |
| `regime` | 当前宏观体制 | `regime` |
| `policy <关键词>` | 政策解读 | `policy 电网十四五规划` |
| `research <问题>` | 自由研究 | `research 如果降息我的组合会怎样` |
| `lookup <关键词>` | 中文名称 / 主题名转 ETF 代码 | `lookup 有色金属ETF代码是多少` |
| `assistant <请求>` | 自然语言路由入口 | `assistant 帮我写今天的晨报 要有路透和彭博的消息` |

## 工作目录

项目根目录：`AI-Finance/`

## 数据依赖

运行前需要配置 `config/config.yaml` 中的 API keys：

- `FRED_API_KEY`：从 [FRED](https://fred.stlouisfed.org/docs/api/api_key.html) 免费申请
- `TUSHARE_TOKEN`：从 [Tushare](https://tushare.pro) 注册获取，可选

## 运行方式

每个命令对应 `src/commands/` 下的同名 Python 文件，通过下面方式执行：

```bash
python -m src.commands.<命令名> <参数>
```
