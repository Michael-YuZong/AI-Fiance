# /briefing

## 输入

- `daily` 或 `weekly`
- 可选 `--news-source <来源>`，例如 `Reuters`、`Bloomberg`

## 执行流程

1. 汇总市场、宏观、持仓和事件数据
2. 先判断“今天市场到底在交易什么”，不要直接把 `regime` 当成主线
3. 对候选主线做交叉校验：
   - 新闻分类是否集中
   - 关键资产价格是否共振
   - 涨停池 / 强势股池 / 龙虎榜是否支持
   - 主力资金 / 北向 / 热门股是否支持
4. 做编辑层裁决：
   - 若今日主线与背景 regime 冲突，由今日主线覆盖日内资产偏好
   - 若价格出现极端值，必须输出异常值复核提示
   - 若新闻源少于 2 类或缺少国内源，必须显式提示覆盖不足
   - 若存在昨日晨报归档，先回顾昨日验证点是否兑现
5. 若本地新闻或行情源不足，先用代理指标补齐主线；必要时允许联网补新闻和事件
6. 生成结构化简报，默认按下面顺序输出：
   - `昨日验证回顾`
   - `主线判断与行动`
   - `资产仪表盘`
   - `驱动与催化`
   - `今日验证点`
   - `组合与持仓`
   - `附录（折叠）`

## 输出

- 晨报或周报 Markdown
- 至少包含：`今日主线`、`今天怎么做`、`宏观资产表`、`Watchlist表`、`核心事件`、`今日日历`、`盘面与资金`、`今日验证点表`、`异常/源覆盖提示`

## 正确性要求

- 如果 `能源/地缘` 新闻很多，且原油大涨、VIX 上行、强势行业集中在电力/电网/油气链，晨报主线应写成 `能源冲击`
- 如果只是 `regime` 偏滞涨或偏复苏，但当天有更强的事件驱动，不允许让 `regime` 覆盖当天主线
- 如果新闻和盘面不一致，要写出“叙事冲突”，而不是给出过度确定的判断
- 如果原油、黄金、ETF 等出现极端涨跌，必须输出 `⚠️ 请人工复核`
- ETF 价格必须注明是 `场内价格`、`现价` 或 `指数点位`
- 如果指数使用 ETF 代理价格，必须明确写出 `代理ETF价格`
- `regime` 判定应尽量使用稳定规则，不应因为是否拿到代理数据而在同一天来回跳
- 不要把同一结论在多个模块重复描述
