# /briefing

## 输入

- `daily` 或 `weekly`
- 可选 `--news-source <来源>`，例如 `Reuters`、`Bloomberg`

## 执行流程

1. 汇总市场、宏观、持仓和事件数据
2. 先判断“今天市场到底在交易什么”，不要直接把 `regime` 当成主线
3. 对候选主线做交叉校验：
   - 新闻分类是否集中
   - 关键资产价格是否共振
   - 涨停池 / 强势股池 / 龙虎榜是否支持
   - 主力资金 / 北向 / 热门股是否支持
4. 若本地新闻或行情源不足，先用代理指标补齐主线；必要时允许联网补新闻和事件
5. 生成结构化简报，并明确区分：
   - 今日主线
   - 背景宏观
   - 主线校验结果

## 输出

- 晨报或周报 Markdown
- 至少包含：`今日主线`、`主线校验`、`重要催化`

## 正确性要求

- 如果 `能源/地缘` 新闻很多，且原油大涨、VIX 上行、强势行业集中在电力/电网/油气链，晨报主线应写成 `能源冲击`
- 如果只是 `regime` 偏滞涨或偏复苏，但当天有更强的事件驱动，不允许让 `regime` 覆盖当天主线
- 如果新闻和盘面不一致，要写出“叙事冲突”，而不是给出过度确定的判断
