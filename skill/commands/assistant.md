# /assistant

## 何时用

- 用户不会记命令
- 用户只会说“分析一下黄金ETF”“帮我写今天的晨报”这类自然语言
- 想先让系统自动判断该走哪条链路

## 最小输入

- 一句自然语言请求
- 可直接带条件，例如“要有路透和彭博的消息”

## 默认行为

1. 先识别是 `晨报 / 分析 / 对比 / 风险 / 回测 / 查代码`
2. 中文主题名先尝试转成代码
3. 自动路由到最匹配的本地命令
4. 用户如果带了 `今天 / 今日 / 现在 / 此刻` 这类时间词，优先路由到带盘中视角的 `scan --today`，必要时补 `snap`
5. ETF / 基金问题优先保留 `基金产品画像`，不要直接退化成泛行业说明
6. 本地命令失败或关键事实缺失时，再联网补洞
7. 仍无法稳定匹配时，回退到 `research`

## 默认输出

- 实际执行的命令
- 简明结论
- 需要时再附完整正文

## 失败时怎么处理

- 标的名不清楚：先给候选，而不是乱绑代码
- 命令能跑但数据缺：说明缺口并补联网结果
- 意图不稳：优先回到 `research`
