# /scan

## 何时用

- 用户明确要分析单个标的
- 需要完整分析，而不是只看一眼行情

## 最小输入

- 标的代码，例如 `561380`、`QQQM`
- 或先经 `assistant / lookup` 解析得到代码

## 默认行为

1. 识别标的类型并拉取历史行情
2. 如果用户问题里包含 `今天 / 今日 / 现在 / 盘中`，补一层 `--today` 盘中快照，并把盘中结论和默认日线框架分开
3. ETF / 基金优先分析 `产品本身`：跟踪基准、前十大持仓、行业暴露、集中度、被动/主动属性
4. 做八维分析或等价的结构化分析
5. 先给一句话结论，再展开驱动、风险、观察点和操作框架
6. 本地源失败时先试备用源，仍不足再联网补足关键事实

## 默认输出

- 一句话结论
- 当前判断
- 今日盘中视角（仅当问题带有“今天/现在/盘中”）
- 基金画像（ETF / 场外基金优先展示）
- 核心驱动
- 风险点
- 观察指标
- 情景分析
- 操作框架

## 失败时怎么处理

- 单个维度缺失：继续分析，但明确标注该维度缺失
- 技术/基本面等核心维度缺失：降低结论强度，不硬下判断
